# Features

## Product Overview

Moonboard Grader helps inspect, predict, generate, and solve Moonboard climbing problems. It combines grid-based grade prediction, movement-sequence analysis, beta solving, synthetic problem generation, analytics, and an interactive web UI.

## Shipped Behavior

### Grid-Based Grade Prediction

The classifier predicts Font grades from Moonboard hold layouts. Problems are represented as 3-channel 18x11 tensors for start, middle, and end holds.

### Backend Prediction API

The FastAPI backend serves model-backed prediction, generation, analytics, problem browsing, health, and board setup routes. It resolves available services by configured hold setup and angle.

### Interactive Frontend

The React frontend renders Moonboard problems, grade predictions, board setup selection, beta views, analytics overlays, duplicate checks, and generated problem workflows.

### Multi-Board And Multi-Angle Support

The app supports configurable hold setups and angles through board setup metadata. Frontend workflows can select the active setup and angle, and backend services resolve model/data capabilities for that combination.

### Beta Solving

The .NET beta solver computes climbing sequences using dynamic programming and move scoring. It is available through both an API and CLI.

### Sequence-Based Classification

The beta classifier predicts grades from move-sequence features generated by the beta solver. This provides an alternative to raw grid-based classification.

### Synthetic Problem Generation

The generator trains and evaluates a conditional VAE that can sample synthetic Moonboard problems for selected difficulty ranges.

### Analytics And Heatmaps

The frontend and backend support analytics-oriented workflows such as hold statistics, heatmaps, attention views, and crux highlighting where the configured data and model artifacts are available.

### Aspire Local Orchestration

The Aspire app host can coordinate the backend, frontend, beta-solver API, and observability dashboard for local full-stack development.

## Planned Behavior

### Moonboard Core Consolidation

Active execution plans track additional consolidation of shared logic into `moonboard_core/`, including:

- Common grid shape constants.
- Safer grade decoding helpers.
- Shared evaluation metrics.
- More complete board configuration validation.
- Reduced duplicate grade, label, split, and raw dataset loading logic across classifier and generator projects.

## Out Of Scope

- Running model training automatically as part of routine agent work.
- Treating generated datasets, checkpoints, logs, or run outputs as hand-authored source.
- Preserving duplicate Moonboard semantics outside `moonboard_core/` when a shared primitive already exists.

## Product Configuration

Board and angle behavior is configured through `config/board_setups.json`. At a product level, that configuration determines which hold setups and wall angles are available, which model/data files back them, and which frontend features can be enabled for a selected setup.
